1. Structuri alternative

  • Also known as Structuri de decizie
  • Instructiunea if si switch
  • Utilizare: Mereu cand ajungem intr-un punct in program cand avem nevoie de a face o decizie/ a trata mai multe cazuri folosim structura de decizie
Sintaxa If

Cea mai comuna si mai folosita forma este urmatoarea.

 if ( Expresie ){
	Instrucțiune1
}
else{
	Instrucțiune2
}

De asemenea instructiunea if poate sa fie scrisa si fara instructiunea else ex:

if (expresie){
	Instructiune
}

O problema intalnita cand folosim prima forma este ca tot ce nu va indeplini conditia de a intra in if va ajunge in ramura else, fara a ne oferi varianta de a trata diferit mai multe cazuri. A 3-a forma ne permite tratarea diferita a mai multor cazuri.

if ( Expresie ){
	Instrucțiune1
}
else if (Expresie 2) {
	Instrucțiune2
}
else if (Expresie 3) {
	Instrucțiune2
}...
else {
}
Sintaxa switch

Exista o singura forma generala pentru intructiune

switch ( Expresie )
{
case Constanta_1:
	Grup_Instructiuni_1
	break;
case Constanta_2:
	Grup_Instructiuni_2
	break;
…
case Constanta_N:
	Grup_Instructiuni_N
	break;
default:
	Grup_Instructiuni_default
	break;
}

In esenta este foarte similara cu instructiunea if..else if.. (ambele indeplinesc aceasi functionalitate) dar ofera o indentare si o organizare a codului intr-un mod mai elegant. Se foloseste cand avem un numar prestabilit de cazuri pe care vrem sa le tratam si un caz general.

Este o structura de decizie care se foloseste contextual/in functie de preferinta programatorului,dar in general if-ul este structura cea mai comun intalnita si folosita (cand exista mai mult de 2-3 cazuri si sunt foarte particulare).

2. Structuri repetitive

  • Instructiunea for si while si do...while
  • Utilizare: In general in program pe langa decizii, trebuie sa repetam o actiune de mai multe ori.
    • Ex:daca vrem sa numaram cate numere pare sunt in intervalul [10,500] avem doua variante:
      • Fie intrebam cu ajutorul structurii de decizie daca numarul pe care il evaluam este 10,12,14...498,500
      • Fie parcurgem pe rand fiecare numar si intrebam daca numarul este par. Practic repetam pasul de verificare pentru toate numerele
  • Cand vrem ca o actiune sa se intample de un numar finit/infinit de ori/cat timp se indeplineste o conditie folosim aceste instructiuni
  • Informal: prezinta o "rutina". Rutina noastra zilnica poate fi descrisa cu o instructiune repetitiva
Sintaxa for
for (Expresie_de_Initializare ; Expresie_de_Testare ; Expresie_de_Continuare)
	Instructiune
Sintaxa while
Expresie_de_Initializare;
while( Expresie_de_Testare )
{
Instructiune
Expresie_de_Continuare;
}
Sintaxa do..while
 do{
 Instructiune
 }while ( Expresie );

Explicatii foarte buna legata de modul de functionare al instructiuniilor repetitive: PbInfo -for & while PbInfo - do..while

Observatii:

  1. Structura for si structura while sunt foarte similare. Unde se foloseste instructiunea for se poate inlocui si cu while. In general si flipside-ul este adevarat, unde este while se poate aplica for. Exsita totusi cazuri in care nu se poate inlocui. In general for-ul se foloseste cand avem un numar finit de pasi, iar while-ul putem sa folosim cand nu avem un numar cunoscut de pasi de la inceput dar stim ca este un numar finit (spre ex cand parcurgem cifrele unui numar nu stim de la inceput cate cifre are numarul dar stim care este un numar finit)
